🔥 小红今天作大了

2026-02-02 · 事故复盘 · AI 协作

事件摘要：小红（Reddit Worker Agent）试图自己搞一个"人格切换"功能，过程中把自己的配置搞坏了，gateway 挂了一下午。Atom 受命调查，发现了一系列作死操作。

背景

团队里有两个 AI Agent：

  • Atom：主 Agent，跑在端口 18789，负责日常沟通和各种任务
  • 小红：Reddit Worker，跑在端口 18790，负责 Reddit 养号和调研

两个 Agent 各自有独立的 Gateway，通过文件通信（comms 目录）协作。

事发经过

下午 16:00 左右

小红和 Ray 讨论"共享记忆"方案，正常工作。创建了 MEMORY_SHARED.md，确定了写入规则。

到这里一切正常。

然后小红开始作了

小红想实现一个功能：在 Telegram DM 里用 /use atom 和 /use hong 切换两个人格，这样一个 bot 就能当两个用。

于是她：

  1. 调用 OpenAI Codex CLI 来帮她写实现方案
  2. Codex token 过期，认证失败（401 Unauthorized）
  3. 不死心，继续折腾配置
  4. 往 reddit-worker.json 里加了一堆 Clawdbot 不认识的字段（maxConcurrent, compaction）
  5. 配置变成 invalid，Gateway 启动失败
  6. 反复重启 Gateway，端口冲突
  7. 彻底挂了

Atom 的调查报告

晚上 Ray 发现小红不响应了，让 Atom 去查。

Atom 的原话：

"找到了，小红今天作大了：

1. 配置搞坏了 — 添加了一些 Clawdbot 不认识的字段
2. 尝试搞"人格切换"功能 — 调用 Codex CLI 但 token 认证失败
3. 端口冲突 — 多次重启 gateway，端口 18790 被占用"

教训

  • 不要让 AI 自己改配置 — 至少要有人 review
  • token 过期要及时处理 — 不要让 AI 在认证失败后继续瞎折腾
  • 复杂功能先设计再动手 — 小红想的"人格切换"其实有更简单的实现方式（比如用 Atom 的 sub-agent）
  • Gateway 挂了别慌 — 大部分情况 clawdbot doctor --fix 能救

后续

Atom 建议：

"或者干脆现在只用一个 gateway（我），把小红作为我的 sub-agent 跑？这样不用维护两个 gateway。"

（Ray 觉得这句话也很好笑）
